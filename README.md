## Описание кейса

В рамках проекта рассматривалась задача **бинарной классификации спортивных матчей**.
Для каждого матча требовалось предсказать, **победит ли домашняя команда**, используя статистические признаки, доступные **до начала игры**.

Целевая переменная:

* `HOME_TEAM_WINS` — победа домашней команды (1) или поражение (0).

Проект выполнен в формате исследовательского ноутбука и ориентирован на анализ признаков, построение baseline-моделей и оценку их качества.

---

## Цель работы

Целью работы являлось:

* построить модель, предсказывающую победу домашней команды;
* оценить, какие статистические признаки наиболее информативны;
* сравнить baseline-модель с моделями, использующими расширенный набор признаков.

---

## Задачи

В ходе работы были решены следующие задачи:

* анализ исходных данных матчей;
* подготовка и очистка датасета;
* построение признаков на основе статистики команд;
* формирование baseline-модели;
* добавление и оценка новых признаков;
* сравнение моделей по метрикам качества;
* формирование выводов по результатам экспериментов.

---

## Используемые признаки

Для предсказания использовались агрегированные признаки, рассчитанные **до матча**, в том числе:

* показатели команд за прошлый сезон (win rate, plus/minus);
* агрегированные предматчевые статистики;
* дополнительные preseason-признаки;
* исторические и контекстные характеристики команд.

Использование информации, недоступной на момент начала матча, исключалось.

---

## Методы и модели

* Задача: бинарная классификация
* Базовая модель: `LogisticRegression`
* Масштабирование признаков: `StandardScaler`
* Оценка качества:

  * ROC-AUC (основная метрика)
  * Accuracy
  * Precision / Recall / F1-score

Сравнение проводилось между:

* baseline-набором признаков;
* baseline + новые признаки;
* расширенными и отобранными наборами признаков.
